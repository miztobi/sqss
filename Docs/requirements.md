# SQSS 汎用型AI試験対策プラットフォーム（PWA版）開発要件定義書
## 兼 一級・二級建築士特化ローンチマスタープラン

本要件定義書は、「過去問から構築する学習計画のサポートツール」をコアの強みとし、開発者自身の合格（0年目）から、将来的に「初年度200人・次年度500人」の有料会員（月額980円）を獲得するための、全資格対応のUGC（ユーザー生成コンテンツ）型総合学習プラットフォームへのスケールを見据えた、システムとコンテンツを完全に分離（疎結合）した次世代アーキテクチャの仕様を規定するものである。

## 1. プロジェクトロードマップ ＆ スケール戦略

### 1.1 フェーズ別の展開計画
* **【フェーズ0】2026年7月〜8月：建築士2冠対策システム完成期**
  * 目標：8月1日の学習スタートに向け、インフラに「Firebase App Hosting」を採用し、SvelteKitのAPI Routes（`+server.js`）でバックエンドを構築。1級建築士および2級建築士の過去問・コラムを「初期埋め込みコンテンツ」として格納し、システムを完成させる。
* **【フェーズ1】2026年8月 〜 2027年7月：実証実験 ＆ クローズドβテスト（次年度無料枠）**
  * 目標：開発者自身が本システムで一級建築士学科に一発合格。同時にYouTube経由で集めた「0年目の戦友（βテスター50名）」に無料解放し、高評価と生々しい合格体験談（UGCの種）を蓄積。初期メンバー50名は、恩義と口コミ最大化のため「次年度（1年間）完全無料（Stripeの12ヶ月100%OFFクーポン制御）」とする。
* **【フェーズ2】2027年8月〜：サービス本格展開（月額980円SaaS）**
  * 目標：合格実績を引っさげ、1級・2級建築士対策アプリとして月額980円で一般公開。初年度200人、次年度500人の有料会員を獲得。
* **【フェーズ3】数年後：UGCインポートによる総合プラットフォームへのスケール**
  * 目標：システム側（UI・AIエンジン）は一切変更せず、ユーザーが自ら「ゴール（資格名）」と「過去問データ（CSV/テキスト）」を自由にアップロードできる機能を解放。あらゆる資格に対応する数千万円〜数億円規模の汎用EdTechプラットフォームへスケールする。

### 1.2 YouTubeショート「生存×時事」ミックス戦略
* **配信手法：** 毎日15〜30秒の縦型ショート動画を、編集コストゼロ（スマホ画面直撮り推奨）で投稿。
* **動画の構成（ミックス）：**
  * **ガワ（新規集客・SEO対策）：** 「最新の法改正ニュース（4号特例廃止など）」や「建築史・名建築の解説」といった、これから勉強を始める新規ユーザーが検索しやすく、目を引く時事ネタを冒頭で提示。
  * **中身（共感・ドキュメンタリー）：** 「残業帰りの夜11時、今夜もこの自作アプリのAI指示通りに30分やる」という泥臭いリアルタイムの成長記録。アプリの「想定点数メーター」が毎日数点ずつ上がっていく様子を可視化し、視聴者に強烈な信頼感と飢餓感を植え付ける。
* **資産化：** あなたが合格した瞬間、過去1年間の動画アーカイブが「このやり方で本当に合格できた」という動かぬ証拠（最強のLP）に化ける。

## 2. コア機能要件：過去問から構築する学習計画サポート

本システムは、機械的な進捗管理ではなく、過去問データからAIがユーザーの「脳内知識マップ」を自動編纂し、日々の行動を完全ナビゲートする。

### 2.1 ベクトルマッピングをベースとしたAI自動タグ付け
* 過去問が登録（インポート）された際、Gemini APIが各設問の文章を解析。表面的な単元名だけでなく、裏で繋がっている概念の相関性（疑似ベクトル空間）を算出し、関連タグ（例：#共用廊下、#容積率緩和）を自動付与する。

### 2.2 動的カリキュラム（学習単元）の構築
* 登録された過去問のボリュームと試験本番までの残り日数から、AIが「週次カリキュラム」を自動生成。ユーザーによる手動での単元追加やスケジュール調整（強度変更）も柔軟に受け入れる設計とする。

### 2.3 隙間ハック：毎日の継続的な過去問 ＆ 時事コラム
* **朝夕10問過去問：** ユーザーの現在の周回モード（1周目：最速、2周目：弱点、3周目：時事予想）に応じて、Firestoreから毎朝夕10問（1日20問）を自動クエリ抽出。片手タップUIでサクサク解かせる。
* **日替わり時事コラム：** 過去問のキーワードと最新ニュースAPIを掛け合わせ、AIが「時事・トレンド」「作品・事例」「基本様式・用語」の3軸コラムを毎朝自動生成（または共通バンクから配信）。画像検索URLを動的生成し、1タップでビジュアル脳内定着を促す。

### 2.4 夜の30分：強度可変型のAI処方特訓（高度インテリジェンス仕様）
* ユーザーが自身の生活に合わせて「今夜の勉強強度（15分・30分・60分）」を選択可能。
* **統計データに基づくAI弱点判定：** アプリ側での機械的な苦手フォルダや特定の箱への転送処理は一切行わない。日中の演習において、フロントエンド（SvelteKit）は単に各設問やタグの「出題回数・正答回数・誤答回数」のカウンターをFirestore上でインクリメント（+1）するだけの軽量処理を担う。
* **リアルタイムプロファイリング：** 夜の特訓開始時、Gemini APIが蓄積された正誤ログの「比率」や「直近の連続誤答数などの推移」を多角的に解析。表面的な間違いの裏にある根本的な「知識のバグ（類似概念の混同）」をAIが自律的に判定・特定し、今夜読むべき法令集のページや概念の整理方法をピンポイントで提案（指示）する。
* **習得度チェックテスト：** 特定されたバグが修正されたかを検証するため、AIがその場で生成、または過去問バンクから動的に抜粋した、今夜限定の一問一答（5〜10問）を解かせる。

## 3. システム・コンテンツ分離（疎結合）アーキテクチャ

将来のUGC化（全資格対応）を見据え、フロントエンド（UI）やAIエンジンは特定の資格（建築士など）のルールに一切依存しない設計とする。試験問題、コラム、ユーザーデータは、すべて「`goalId`（資格識別ID）」を最上位に持つ階層構造で管理する。

### 📊 アーキテクチャ概念構造
フロントエンドからバックエンド、データベースにいたるまでのデータ分離モデルは以下の階層の通りに定義する。

> [ SvelteKit フロントエンド PWA (共通UI / 音声・画像検索ラッパー) ]
>                       │
>                       ▼ API Routes (共通AIプロファイリング・エンジン)
> ┌──────────────────────────────────────────────────────────────┐
> │                  Firebase App Hosting                        │
> │ ┌──────────────────────────────────────────────────────────┐ │
> │ │ Cloud Firestore (疎結合マルチテナント型スキーマ)           │ │
> │ │                                                          │ │
> │ │  goals/ (すべてのコンテンツはここに内包)                 │ │
> │ │   ├── 1kyu_kenchikushi/ (フェーズ0：埋め込みコンテンツ)   │ │
> │ │   │    ├── past_questions/                               │ │
> │ │   │    └── daily_columns/                                │ │
> │ │   └── user_uploaded_id_999/ (フェーズ3：ユーザーUGC)    │ │
> │ │        ├── past_questions/                               │ │
> │ │        └── daily_columns/                                │ │
> │ └──────────────────────────────────────────────────────────┘ │
> └──────────────────────────────────────────────────────────────┘

* **フロントエンド ＆ バックエンド（API）：** SvelteKit / Tailwind CSS
  * バックエンド処理（Gemini API呼び出し、Stripe連携、Firestore直操作）はすべてSvelteKitの「API Routes（`+server.js`）」に記述し、プロジェクトを一元化する。
  * PWA（Progressive Web App）化を必須要件とし、Safari/Chromeの「ホーム画面に追加」を促すUIを実装。
  * Service WorkerによるPush APIを活用し、毎朝夕の過去問配信リマインドを「永年無料」で実装。
* **インフラ・ホスティング：** Firebase App Hosting
  * SvelteKitのSSRおよびAPI RoutesをネイティブサポートするFirebaseの次世代ホスティングを採用。サーバーレス関数のタイムアウト上限が自動拡張されるため、Geminiの重いプロファイリング処理も確実に実行可能。
* **データベース ＆ 認証：** Cloud Firestore / Firebase Authentication
* **AIエンジン：** Gemini API (無料枠内運用、JSON構造化出力によるAPI消費の最小化)。
* **決済システム：** Stripe（ストライプ）

## 4. 画面（ページ）構成 ＆ 各画面の機能要件

スマホ片手での「爆速 scannable（見やすさ）」と「ゲーミフィケーション」を徹底した4画面構成。

### 4.1 ダッシュボード画面（トップページ）
* **想定点数メーター（最上部）：**
  「直近1ヶ月の各単元の正解率」×「本試験の配点ウエイト」をSvelteKitのサーバーサイドでリアルタイム算出し、「もし今日、本試験を受けたら何点取れるか（125点満点）」を模試のように表示。
* **全国ライバル比較図（ベルカーブ）：** 全ユーザーの想定点数の平均値を中心としたグラフ内に、自分の位置を視覚化。
* **カウントダウン ＆ 今週のノルマ達成度バー：** 今週のノルマ50問中、現在の消化数をProgressタグを用いてダイナミックに表示。
* **単元別・戦闘マップ（ツリーマップ風）：** CSS Gridを使用。タイルの「大きさ」が出題ウエイトを表し、「色」がAIの判定した定着度（緑：安全、赤：弱点、グレー：未履修）を表す。

### 4.2 過去問演習・一問一答画面
* **中断・再開機能：** 解答状態を一時保存。
* **朝夕10問ずつの自動切り替え：** 朝は「午前10問」、夕方は「午後の10問」が、現在の周回フラグに基づき自動クエリ抽出され、片手タップUIでセットされる。

### 4.3 夜の30分・AI処方特訓画面
* 前述（2.4）の強度可変型AI処方特訓を実行する中核画面。選択された強度時間（15分〜60分）のタイマーとともに、AIがプロファイリングしたバグの診断テキストおよび専用の検証テストが表示される。

### 4.4 過去ログ・カリキュラム確認画面
* 土曜日の実戦テストを受けるための画面。AIによる週次・月次の総括レポート、および過去のアーカイブを閲覧可能。

## 5. 柔軟な受講プラン要件（途中参加 ＆ 2年計画）

* **途中参加ユーザー（残り日数逆算型自動スケジューリング）：**
  本試験までの残り日数からAIが自動逆算。目標周回数や1日のノルマ問数を動的に再計算して最短ルートを提供。
* **再来年合格を目指すユーザー（2年合格プラン / 全110週間）：**
  「超・超・丁寧進行モード」を標準装備。1週間の美しいサイクルはそのまま維持し、日々の過去問ノルマを「1日5問〜10問」に極限まで落とす。

コードブロックのネストを避けるため、各ドキュメントのJSON構造は以下に引用符を用いて記述する。

>   "createdAt": "2026-07-01T00:00:00Z",
>   "planStatus": "premium_free_year", // beta_testing, premium_active, premium_free_year
>   "freePeriodEndsAt": "2028-07-31T23:59:59Z",
>   "stripeSubscriptionId": "sub_xxxxxx",
>   "studyPlanType": "1_year_pass"
> }

>   "currentWeek": 1,
>   "currentLoop": 1,
